<contract>
Контракт spacing:
- Это **gap между соседними элементами**, НЕ внешний padding (перед первым и после последнего spacing не добавляется).
- Для count элементов с размерами s_0..s_(count-1):
  - GetItemOffset(i) = sum(s_0..s_(i-1)) + i*spacing  (для i=0 → 0)
  - TotalSize = sum_all + max(0, count-1)*spacing
- Дефолт spacing = 0f → формулы вырождаются в текущие, существующие тесты должны проходить без изменений.
- Ось-агностично: LayoutCalculator оперирует скалярами; spacing применяется одинаково для Y и X.
</contract>

    - LayoutCalculator.InsertAt/RemoveAt: при пересчёте prefix sum от index до конца — spacing добавляется так же (для i>0).
    - LayoutCalculator.GetItemOffset(index) для fixed-path: возвращает `index * (_fixedHeight + _spacing)` (даёт `i*fixedHeight + i*spacing`, что равно требуемому offset).
    - LayoutCalculator.TotalHeight для fixed-path: возвращает `_itemCount > 0 ? _itemCount * _fixedHeight + (_itemCount - 1) * _spacing : 0f`.
    - LayoutCalculator.GetItemHeight(index) для variable-path: `_prefixHeights[i+1] - _prefixHeights[i] - (i > 0 ? _spacing : 0)`. Для i=0: `_prefixHeights[1] - _prefixHeights[0]`. Для fixed-path остаётся `_fixedHeight`.
    - LayoutCalculator.FindVisibleRange: variable-path работает без изменений (binary search по prefix sum уже учитывает spacing). Fixed-path: stride = `_fixedHeight + _spacing`; `firstVisible = (int)(scrollPosition / stride)`, `lastVisible = (int)((scrollPosition + viewportHeight) / stride)`. При spacing=0f stride=fixedHeight, поведение идентично текущему. Важно: защита от деления на ноль — если stride <= 0f, возвращать range из одного элемента (текущая логика и так не работает с zero fixedHeight).
    - VirtualCollectionBinding: в `RebuildLayout(int count)`, `OnElementAdded`, `OnElementRemoved` ПЕРЕД вызовом `_layoutCalculator.Rebuild/InsertAt/RemoveAt` вызывать `_layoutCalculator.SetSpacing(_scrollRect.Spacing)`. Достаточно одной точки — в начале `RebuildLayout`, и в начале `OnElementAdded`/`OnElementRemoved` (там вызов InsertAt/RemoveAt напрямую). По сути spacing нужно поддерживать в актуальном состоянии до любой операции, меняющей prefix sum.
