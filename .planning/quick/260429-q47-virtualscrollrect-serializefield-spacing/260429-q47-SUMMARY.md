# Quick 260429-q47: VirtualScrollRect SerializeField spacing Summary

**One-liner:** Добавлено SerializeField-поле `_spacing` в VirtualScrollRect и проброс в LayoutCalculator через VirtualCollectionBinding, ось-агностичная вставка зазора между соседними элементами с сохранением полной обратной совместимости при spacing=0f.

## Что сделано

### Task 1: Runtime — spacing в VirtualScrollRect, LayoutCalculator, VirtualCollectionBinding

**VirtualScrollRect.cs:**
- Новое `[SerializeField] private float _spacing = 0f;` (рядом с `_overscanCount`)
- `internal float Spacing => _spacing;` (рядом с `OverscanCount`)

**LayoutCalculator.cs:**
- Новое поле `private float _spacing;`
- Метод `public void SetSpacing(float spacing)` — устанавливает зазор без ребилда
- **Семантика prefix sum изменена**: `prefix[i]` теперь хранит **offset элемента i**, а не «конец i-го элемента». То есть `prefix[i+1] - prefix[i] = s_i + _spacing` для всех i (один stride: элемент + зазор после). Это упрощает `GetItemOffset(i) = prefix[i]` (без поправки) и согласуется с контрактом плана `offset(i) = sum(s_0..s_(i-1)) + i*spacing`.
- `TotalHeight` для variable-path: `prefix[N] - _spacing` (исключаем trailing spacing после последнего элемента).
- `TotalHeight` для fixed-path: `_itemCount * _fixedHeight + (_itemCount - 1) * _spacing`.
- `Rebuild(int, float)` и `Rebuild(int, Func)` дополнены: `_prefixHeights[i+1] = _prefixHeights[i] + size + _spacing` для всех i.
- `InsertAt`/`RemoveAt`: та же индукция, `prefix[index]` остаётся корректным (offset элемента не зависит от элементов справа).
- `GetItemOffset` для fixed-path: `index * (_fixedHeight + _spacing)`.
- `GetItemHeight` для variable-path: `prefix[i+1] - prefix[i] - _spacing` (raw size без зазора).
- `FindVisibleRange` для fixed-path: stride = `_fixedHeight + _spacing`, защита от деления на ноль (если stride ≤ 0 — возвращает весь диапазон). Variable-path не изменился — binary search по prefix sum уже работает корректно с новой семантикой.

**VirtualCollectionBinding.cs:**
- В `RebuildLayout(int count)`, `OnElementAdded`, `OnElementRemoved` ПЕРЕД вызовом `Rebuild/InsertAt/RemoveAt` добавлен `_layoutCalculator.SetSpacing(_scrollRect.Spacing)` — гарантирует синхронизацию при изменении значения в инспекторе.

## Контракт spacing

- **Зазор между соседними элементами**, НЕ внешний padding (перед первым и после последнего spacing не добавляется).
- Для count элементов с размерами s_0..s_(count-1):
  - `GetItemOffset(i) = sum(s_0..s_(i-1)) + i * spacing` (для i=0 → 0)
  - `TotalSize = sum_all + max(0, count-1) * spacing`
- При spacing=0f формулы вырождаются в исходные — все существующие тесты проходят без изменений.
- Ось-агностично: `LayoutCalculator` оперирует скалярами; spacing применяется одинаково для Y и X.

## Deviations from Plan

### Расхождение 1: prefix-sum semantics adjustment

**Найдено во время:** Task 1 (после первой версии правок)

**Issue:** План предлагал индукцию prefix-sum в стиле «add = size + (i>0 ? spacing : 0)», что давало `prefix[i] = "конец i-го элемента"` (offset(i+1) - spacing). Это конфликтовало с прямым контрактом плана `GetItemOffset(i) = sum + i*spacing`: при моей первой реализации `GetItemOffset(1)` для heights={50,100,...} spacing=10 возвращал 50, а план требовал 60.

**Fix:** Переключил семантику prefix sum на «`prefix[i] = offset элемента i`» (включая spacing перед элементом, кроме i=0). Индукция стала проще: `prefix[i+1] = prefix[i] + size + spacing` для всех i. `TotalHeight` корректируется на `prefix[N] - spacing`, чтобы исключить trailing spacing после последнего элемента. Это математически эквивалентно плановому контракту и одинаково корректно работает для существующих тестов (при spacing=0 формула вырождается).

## Edge Cases

- **stride ≤ 0f в FindVisibleRange fixed-path:** при `_fixedHeight + _spacing ≤ 0` возвращаем range [0..N-1] (весь список). Это защищает от деления на ноль (например, если будущий код выставит negative spacing == -fixedHeight). Текущая логика и так не работала с zero fixedHeight — поведение не ухудшается.
- **InsertAt при index=0:** prefix-sum полностью пересчитывается с нуля, формула `prefix[i+1] = prefix[i] + size + spacing` для всех i = index..newCount-1 даёт корректный результат — `prefix[0]=0`, далее offset элементов 1, 2, …
- **Variable-path FindVisibleRange:** binary search по prefix sum работает без изменений, потому что новая семантика `prefix[mid+1] = offset элемента (mid+1)` точнее старой («конец mid»), и `prefix[mid+1] <= scrollPosition` означает «scroll прошёл начало элемента mid+1» → ищем индекс, который ещё «не виден» → корректный first/last.
- **Trailing spacing:** В контракте «зазор только между элементами» — `TotalHeight` корректирует на `-spacing`, иначе scroll-area стал бы на один зазор больше, чем нужно.
