# Phase 260501-wq7 Plan 01: Multi-select toggle in ViewModelViewerWindow Summary

**One-liner:** HashSet + per-VM Dictionary заменили одиночный _selectedViewModel, toggle теперь аддитивен.

## What Was Done

Заменена одиночная single-selection модель `_selectedViewModel` / `_displayedViewModel` / `_viewModelDrawer`
на multi-selection: `HashSet<AbstractViewModel> _selectedViewModels` и `Dictionary<AbstractViewModel, ViewModelDrawer> _drawerPerViewModel`.

Ключевые изменения в `Editor/ViewModelViewerWindow.cs` (коммит `0ccff1b`):

| Элемент | До | После |
|---|---|---|
| Хранение выбора | `_selectedViewModel` (одно поле) | `_selectedViewModels` (HashSet) |
| Drawer | `_viewModelDrawer` (один, stateful) | `_drawerPerViewModel` (словарь, свежий на каждый rebuild) |
| Toggle callback | присваивает поле при `true`, игнорирует `false` | Add при `true`, Remove+dispose при `false` |
| `RebuildViewModelDisplay` | рендерит один ViewModel | итерирует все выбранные, создаёт drawer на каждый |
| `OnEditorUpdate` | `UpdateValues()` на одном drawer'е | `UpdateValues()` на всех drawer'ах из словаря |
| `RebuildSelector` | сбрасывал поле на `null` | очищает HashSet/Dictionary от stale VM |
| `ShowUnsupportedFields` | сеттер на одном drawer'е | поле `_showUnsupportedFields`, передаётся каждому drawer'у при rebuild |
