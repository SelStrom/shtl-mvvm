**Диагноз (уже подтверждён чтением файла):**

- Строка 22: `private AbstractViewModel _selectedViewModel;` — одиночное поле, перезаписывается при каждом toggle.
- Строка 23: `private AbstractViewModel _displayedViewModel;` — только один отображаемый.
- Строка 26: `private readonly ViewModelDrawer _viewModelDrawer;` — один drawer, stateful (при вызове `BuildViewModelElement` делает `_valueUpdaters.Clear()`), не поддерживает несколько ViewModel одновременно.
- Строки 133–139: toggle callback делает `_selectedViewModel = capturedWidget` — одиночное присваивание.
- Строки 89–103: `OnEditorUpdate` сравнивает `_selectedViewModel != _displayedViewModel` — вся логика обновления заточена под один.
- Строка 160: `RebuildViewModelDisplay` рендерит только `_selectedViewModel`.

**Изменения:**

1. **Удалить** поля `_selectedViewModel`, `_displayedViewModel` и `_viewModelDrawer` (одиночный).

2. **Добавить** поля:
   ```csharp
   private readonly HashSet<AbstractViewModel> _selectedViewModels = new();
   private readonly Dictionary<AbstractViewModel, ViewModelDrawer> _drawerPerViewModel = new();
   ```

3. **Конструктор** `ViewModelViewerWindow()`: удалить инициализацию `_viewModelDrawer`, она теперь не нужна.

4. **`RebuildSelector`**: изменить toggle callback — вместо присвоения одиночного поля использовать Add/Remove в HashSet. Также исправить логику сброса: при изменении списка виджетов удалять из `_selectedViewModels` и `_drawerPerViewModel` ViewModel-ы, которых больше нет в `widgets`.
   ```csharp
   private void RebuildSelector(List<AbstractViewModel> widgets)
   {
       _selectorContainer.Clear();

       // Remove stale entries no longer present in scene
       _selectedViewModels.RemoveWhere(vm => !widgets.Contains(vm));
       foreach (var stale in _drawerPerViewModel.Keys.Where(vm => !widgets.Contains(vm)).ToList())
       {
           _drawerPerViewModel.Remove(stale);
       }

       if (widgets.Count <= 0)
       {
           RebuildViewModelDisplay();
           return;
       }

       foreach (var widget in widgets)
       {
           var isSelected = _selectedViewModels.Contains(widget);
           var capturedWidget = widget;
           var widgetToggle = new Toggle(widget.GetType().Name) { value = isSelected };
           widgetToggle.RegisterValueChangedCallback(evt =>
           {
               if (evt.newValue)
               {
                   _selectedViewModels.Add(capturedWidget);
               }
               else
               {
                   _selectedViewModels.Remove(capturedWidget);
                   _drawerPerViewModel.Remove(capturedWidget);
               }
               RebuildViewModelDisplay();
           });
           _selectorContainer.Add(widgetToggle);
       }

       if (widgets.Count == 1)
       {
           _selectedViewModels.Add(widgets[0]);
           RebuildViewModelDisplay();
       }
   }
   ```

5. **`OnEditorUpdate`**: убрать сравнение `_selectedViewModel != _displayedViewModel`. Вместо этого — итерировать по `_selectedViewModels` и вызывать `UpdateValues()` для каждого drawer'а. Если хотя бы один возвращает `true` — перестроить display.
   ```csharp
   private void OnEditorUpdate()
   {
       if (_viewModelContainer == null)
       {
           return;
       }

       UpdateSelector();

       if (_selectedViewModels.Count == 0)
       {
           return;
       }

       var needsRebuild = false;
       foreach (var vm in _selectedViewModels)
       {
           if (_drawerPerViewModel.TryGetValue(vm, out var drawer) && drawer.UpdateValues())
           {
               needsRebuild = true;
           }
       }

       if (needsRebuild)
       {
           RebuildViewModelDisplay();
       }
   }
   ```
   Убрать также строки 89–98 с проверкой `_selectedViewModel != _displayedViewModel` — этот guard больше не нужен, т.к. `RebuildSelector` сам вызывает `RebuildViewModelDisplay()` при изменениях.

   **Замечание:** Чтобы не вызывать `RebuildViewModelDisplay()` каждый тик когда набор не изменился, drawer'ы для уже выбранных ViewModel создаются в `RebuildViewModelDisplay()` и кешируются в `_drawerPerViewModel`. При повторном вызове `RebuildViewModelDisplay` нужно **пересоздавать** drawer'ы (как это делалось раньше), потому что UIElements-дерево перестраивается с нуля — иначе `_valueUpdaters` в drawer'е будут указывать на несуществующие элементы.

6. **`RebuildViewModelDisplay`**: итерировать по `_selectedViewModels`, для каждого создавать свежий `ViewModelDrawer` (showUnsupportedFields берём из toggle в окне), строить элемент и добавлять в контейнер. Кешировать drawer в `_drawerPerViewModel`.
   ```csharp
   private void RebuildViewModelDisplay()
   {
       _viewModelContainer.Clear();
       _drawerPerViewModel.Clear();

       if (_selectedViewModels.Count == 0)
       {
           _viewModelContainer.Add(new HelpBox("No active view models found", HelpBoxMessageType.Info));
           return;
       }

       foreach (var vm in _selectedViewModels)
       {
           var drawer = new ViewModelDrawer(false)
           {
               ShowUnsupportedFields = _showUnsupportedFields
           };
           _drawerPerViewModel[vm] = drawer;
           _viewModelContainer.Add(drawer.BuildViewModelElement(vm.GetType(), vm));
       }
   }
   ```

7. **`ShowUnsupportedFields` state**: Сейчас `_viewModelDrawer.ShowUnsupportedFields` устанавливается в toggle callback (строка 49). Нужно добавить поле `private bool _showUnsupportedFields;` в класс и обновлять его в toggle callback вместо обращения к одиночному drawer'у:
   ```csharp
   toggle.RegisterValueChangedCallback(evt =>
   {
       _showUnsupportedFields = evt.newValue;
       RebuildViewModelDisplay();
   });
   ```
