# Quick 260427-r28: VirtualScrollRect ScrollAxis + PositionView axis branching Summary

Добавлена поддержка горизонтальной оси скролла в виртуализированном списке без
ломания обратной совместимости — существующие вертикальные сцены и тесты работают
без изменений (default ось = Vertical).

## Что сделано

### Task 1 — `094cd92`
- В `VirtualScrollRect` добавлен `internal enum ScrollAxis { Vertical, Horizontal }`
  и `[SerializeField] private ScrollAxis _axis = ScrollAxis.Vertical` (обратная совместимость)
- Новое свойство `internal ScrollAxis Axis => _axis` — биндинг читает ось через него
- `ViewportHeight` → `ViewportSize` (Vertical: rect.height, Horizontal: rect.width)
- `OnDrag`: drag-delta берётся по выбранной оси, знак для Horizontal противоположный
  (свайп вправо двигает viewport вправо по контенту)
- `OnScroll`: для Horizontal используется `scrollDelta.x`, fallback на `.y` если `.x == 0`
  (как Unity ScrollRect)
- `GetLocalPosition` ветвится по оси (`localPoint.x` или `localPoint.y`)
- `UpdateScrollbar` использует общий `ViewportSize`; `SetActive(needsScroll)` на gameObject
  Scrollbar остаётся (закрывает требование скрытия области трэка целиком)
- В `VirtualCollectionBinding.UpdateVisibleRange` обновлён вызов `_scrollRect.ViewportSize`
- В `LayoutCalculatorTests` добавлены ось-агностичные тесты:
  - `FindVisibleRange_AxisAgnostic_FixedSize_SameResultForXAndY`
  - `FindVisibleRange_AxisAgnostic_VariableSize_SameResultForXAndY`

### Task 2 — `f932333`
- `VirtualCollectionBinding.PositionView` ветвится по `_scrollRect.Axis`:
  - Vertical (без изменений): anchors=(0,1)/(1,1), pivot=(0,1),
    `anchoredPosition = (0, -(offset - scrollPosition))`,
    `sizeDelta = (rt.sizeDelta.x, itemHeight)`
  - Horizontal: anchors=(0,0)/(0,1), pivot=(0,0),
    `anchoredPosition = (offset - scrollPosition, 0)`,
    `sizeDelta = (itemWidth, rt.sizeDelta.y)`
- Изменений в публичном API нет (`Bind.From().To()` и `ReactiveVirtualList` без изменений)
- В `VirtualCollectionBindingTests`:
  - вынесен helper `ConfigureScrollRect(ScrollAxis axis, Vector2 viewportSize)`,
    выставляющий `_viewport` и `_axis` через рефлексию
  - `OnContentChanged_HorizontalAxis_RebuildLayoutAndCreatesViews`
  - `ScrollPositionChange_HorizontalAxis_UpdatesVisibleViews`
  - `HorizontalAxis_FirstVisibleIndex_IsZeroAtScrollZero`

## Контракты, которые НЕ изменились

- Публичный API пакета: `Bind.From(...).To(...)`, `ReactiveVirtualList<T>`,
  `VirtualListBindExtensions` — без изменений сигнатур
- `LayoutCalculator` — внутренние методы и публичные свойства как были (включая `TotalHeight`)
- Поведение Vertical-сцен — идентично прежнему (default `ScrollAxis.Vertical`)

## Task 3 — Manual Verification (human-verify, defer to user)

Эта задача требует ручной проверки в Unity Editor. Автоматическая часть
(coding + unit tests) выполнена в Task 1 и Task 2; следующее должен проверить
пользователь в `Samples~/Sample/Assets/Scenes/mvvm_demo.unity`:

### Сценарий A — Vertical (regression check)

1. Открыть scene `mvvm_demo.unity`, перейти на экран VirtualList
2. Запустить Play Mode
3. Проверить:
   - При пустом списке `gameObject` Scrollbar полностью скрыт (область трэка
     не занимает места в layout)
   - При добавлении достаточного числа элементов Scrollbar появляется и работает
   - Скролл колесом, drag и Scrollbar-handle двигают контент в одном направлении
     (как до правок)

### Сценарий B — Horizontal (новая ось)

1. На demo-префабе `VirtualScroll` переключить `_axis` на `Horizontal` в инспекторе
2. Назначить горизонтальный Scrollbar в поле `_scrollbar`
3. Прогнать те же три проверки по X:
   - Drag по X двигает контент по X
   - Scroll wheel прокручивает (обычное колесо или Shift+wheel — оба должны работать
     благодаря fallback на `scrollDelta.y`)
   - Scrollbar.gameObject скрывается, когда контент уже viewport, появляется,
     когда контент шире viewport

### Сценарий C — Layout integrity при скрытии Scrollbar

Если в сцене стоит `VerticalLayoutGroup` / `HorizontalLayoutGroup` рядом с
Viewport и Scrollbar — после скрытия `Scrollbar.gameObject` layout должен
перераспределить место (область трэка не торчит пустым гэпом).

### Если сценарий B не работает

Сообщить фактическое поведение:
- Что не двигается?
- Куда уезжает контент?
- Какой `Scrollbar.direction` в сцене?
