       `override Activate()`:
       - Инициализация LayoutCalculator (capacity = vmList.Count + 16 для запаса)

       `OnContentChanged(ReactiveList<TViewModel> list)`:
       - Rebuild LayoutCalculator с текущими данными
       - SetContentHeight на scrollRect
       - _isDirty = true
       - UpdateVisibleRange()

       `OnElementAdded(int index, TViewModel element)`:
       - _layoutCalculator.InsertAt(index, _vmList.Count, ...)
       - _scrollRect.SetContentHeight(_layoutCalculator.TotalHeight)
       - Если index < _currentRange.FirstIndex -- корректировка scroll position: `_scrollRect.ScrollPosition += _vmList.GetItemHeight(index)` (per UI-SPEC)
       - Пересдвинуть индексы в _activeViews для элементов >= index (shift +1)
       - UpdateVisibleRange()

       `OnElementRemoved(int index, TViewModel element)`:
       - Если index в _activeViews -- Release View в pool
       - _layoutCalculator.RemoveAt(index, _vmList.Count, ...)
       - _scrollRect.SetContentHeight(_layoutCalculator.TotalHeight)
       - Если index < _currentRange.FirstIndex -- корректировка scroll position: `_scrollRect.ScrollPosition -= heightOfRemoved`
       - Пересдвинуть индексы в _activeViews для элементов > index (shift -1)
       - UpdateVisibleRange()

       `OnElementReplaced(int index, TViewModel element)`:
       - Если index в _activeViews: view.Dispose(); view.Connect(element) (per Pitfall 3: Dispose перед Connect)
       - _isDirty = true (высота могла измениться)
       - RebuildLayout + UpdateVisibleRange

       `OnScrollPositionChanged(float scrollPosition)`:
       - Синхронизация с ReactiveVirtualList: `_vmList.ScrollPosition.Value = scrollPosition`
       - UpdateVisibleRange()

       `UpdateVisibleRange()` -- КЛЮЧЕВОЙ метод, вызывается при скролле и мутации:
       - `var newRange = _layoutCalculator.FindVisibleRange(_scrollRect.ScrollPosition, _scrollRect.ViewportHeight, _scrollRect.OverscanCount)`
       - Определить элементы, вышедшие из диапазона: foreach index in _activeViews.Keys -- если index < newRange.FirstIndex || index > newRange.LastIndex -- Release(view)
       - Определить новые элементы: for index from newRange.FirstIndex to newRange.LastIndex -- если index не в _activeViews -- Get view из pool, Connect(vmList[index]), позиционировать
       - Позиционирование View: `view.GetComponent<RectTransform>().anchoredPosition = new Vector2(0, -_layoutCalculator.GetItemOffset(index) + _scrollRect.ScrollPosition)` -- НЕТ, позиция должна быть абсолютной относительно content, а не viewport. Правильно: `anchoredPosition = new Vector2(0, -_layoutCalculator.GetItemOffset(index))`. Viewport offset обеспечивается через scroll position content parent.

         ВАЖНО: Элементы позиционируются как дочерние объекты content-контейнера. Content-контейнер сдвигается на scrollPosition. RectTransform элемента: anchor top-left, pivot (0,1), anchoredPosition.y = -_layoutCalculator.GetItemOffset(index), sizeDelta.y = _layoutCalculator.GetItemHeight(index).
       - Обновить `_currentRange = newRange`
       - Обновить viewport state: `_vmList.FirstVisibleIndex.Value = newRange.FirstIndex; _vmList.VisibleCount.Value = newRange.Count`
