<task type="auto">
  <name>Task 1: Исправить LateUpdate гейт и velocity stop threshold (Gap 2 — Elastic bounce)</name>
  <files>Runtime/Core/VirtualScroll/VirtualScrollRect.cs</files>
  <action>
Прочитать Runtime/Core/VirtualScroll/VirtualScrollRect.cs целиком.

**Проблема (Gap 2):** LateUpdate (строка 232) делает early-return при `_velocity == 0f` без учёта `CalculateOffset()`. Это блокирует Elastic-возврат после mouse wheel, который сдвигает позицию без установки velocity. W-04 fix (SetContentSize обнуляет velocity при out-of-range) также создаёт состояние out-of-range + velocity==0 → Elastic не запускается.

Дополнительно: порог остановки velocity (строки 260-263) обнуляет velocity при `Mathf.Abs(_velocity) < 1f` без проверки offset → прерывает Elastic-цикл до достижения границы.

**Изменение 1: LateUpdate гейт (строка 232)**

Заменить:
```csharp
if (_isDragging || _velocity == 0f)
{
    return;
}

var offset = CalculateOffset();
```

На:
```csharp
if (_isDragging)
{
    return;
}

var offset = CalculateOffset();
if (_velocity == 0f && offset == 0f)
{
    return;
}
```

Логика: early-return только если нет движения И нет offset для Elastic-возврата. Если offset != 0, LateUpdate должен продолжить выполнение для SmoothDamp, даже при velocity==0.

**Изменение 2: Velocity stop threshold (строки 260-263, после обоих ветвей if/else if/else)**

Заменить:
```csharp
if (Mathf.Abs(_velocity) < 1f)
{
    _velocity = 0f;
}
```

На:
```csharp
if (Mathf.Abs(_velocity) < 1f && offset == 0f)
{
    _velocity = 0f;
}
```

Логика: обнулять velocity только когда она близка к нулю И позиция уже в допустимом диапазоне. Без этого SmoothDamp может быть прерван раньше, чем позиция достигнет boundaries.

**Важно:** переменная `offset` уже вычислена в начале метода после изменения 1, поэтому второй вызов `CalculateOffset()` не нужен — использовать ту же переменную.

**Стиль:** Allman braces (скобки на новой строке), 4 пробела. Комментарий к изменённой логике на русском:
```csharp
// Прерываем цикл только если нет ни инерции, ни offset для Elastic-возврата.
// velocity==0 без offset — действительно остановились; velocity==0 с offset — нужен SmoothDamp.
```

Убедиться, что `offset` используется в блоке velocity stop threshold — это та же переменная из начала LateUpdate, не новый вызов метода.

<task type="auto">
  <name>Task 2: Добавить guard для короткого списка в OnBeginDrag, OnDrag, OnScroll (Gap 3)</name>
  <files>Runtime/Core/VirtualScroll/VirtualScrollRect.cs</files>
  <action>
Прочитать текущее состояние Runtime/Core/VirtualScroll/VirtualScrollRect.cs (после Task 1).

**Проблема (Gap 3):** OnDrag и OnScroll безусловно модифицируют `_scrollPosition`, не проверяя что контент вообще требует скролла. Когда `_contentHeight <= ViewportSize`, maxScroll == 0 — скроллить некуда. В Elastic режиме позиция временно уплывает, LateUpdate её возвращает → пользователь видит rubber-band вместо полной блокировки.

**Изменение 1: OnBeginDrag (строка 175)**

Добавить в начало метода, ДО всех других инструкций:
```csharp
public void OnBeginDrag(PointerEventData eventData)
{
    // Скролл недоступен — контент помещается во viewport без прокрутки
    if (_contentHeight <= ViewportSize)
    {
        return;
    }

    _isDragging = true;
    _velocity = 0f;
    _prevDragPosition = GetLocalPosition(eventData);
}
```

Обоснование: если OnBeginDrag не устанавливает `_isDragging = true`, то OnDrag/OnEndDrag также не получат смысловой контекст drag, и если guard в OnDrag присутствует, это ещё более чистое решение. Главное — _isDragging остаётся false, поэтому LateUpdate не будет блокироваться на `_isDragging` check.

**Изменение 2: OnDrag (строка 182)**

Добавить guard в начало метода:
```csharp
public void OnDrag(PointerEventData eventData)
{
    // Скролл недоступен — контент помещается во viewport без прокрутки
    if (_contentHeight <= ViewportSize)
    {
        return;
    }

    var pos = GetLocalPosition(eventData);
    // ... остальная логика без изменений
```

**Изменение 3: OnScroll (строка 208)**

Добавить guard в начало метода:
```csharp
public void OnScroll(PointerEventData eventData)
{
    // Скролл недоступен — контент помещается во viewport без прокрутки
    if (_contentHeight <= ViewportSize)
    {
        return;
    }

    float rawDelta;
    // ... остальная логика без изменений
```

**Стиль:** Allman braces, 4 пробела, комментарий на русском перед guard (одинаковый во всех трёх местах).

**Zero-alloc:** guard `_contentHeight <= ViewportSize` — чистое сравнение float, ноль аллокаций. ViewportSize — AggressiveInlining property, доступен без аллокации.
