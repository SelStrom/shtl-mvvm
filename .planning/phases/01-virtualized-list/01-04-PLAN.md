<task type="auto">
  <name>Task 1: Code review на zero-alloc hot path</name>
  <files></files>
  <read_first>
    - Runtime/Core/VirtualScroll/VirtualScrollRect.cs (LateUpdate, OnDrag, OnScroll -- hot path)
    - Runtime/Core/Bindings/VirtualCollectionBinding.cs (UpdateVisibleRange -- hot path при каждом скролле)
    - Runtime/Core/VirtualScroll/ViewRecyclingPool.cs (Get, Release -- вызываются при recycling)
    - Runtime/Core/VirtualScroll/LayoutCalculator.cs (FindVisibleRange -- вызывается при каждом скролле)
  </read_first>
  <action>
    Провести code review всех hot path методов на предмет heap-аллокаций:

    **Checklist (каждый пункт должен быть OK):**
    1. VirtualScrollRect.LateUpdate: нет `new`, нет LINQ, нет string interpolation, нет boxing
    2. VirtualScrollRect.OnDrag: нет `new` кроме struct Vector2 (stack), нет LINQ
    3. VirtualScrollRect.OnScroll: нет `new`, нет LINQ
    4. VirtualCollectionBinding.UpdateVisibleRange: нет `new`, нет LINQ, нет lambda, _indicesToRemove предаллоцирован
    5. ViewRecyclingPool.Get: нет `new` при непустом пуле, только Instantiate при создании нового View
    6. ViewRecyclingPool.Release: нет `new`, нет LINQ
    7. LayoutCalculator.FindVisibleRange: нет `new`, возвращает struct VisibleRange (stack), нет LINQ
    8. Dictionary<int, TWidgetView> итерация: struct enumerator (OK для Dictionary)
    9. Нет foreach по IEnumerable (только по конкретным коллекциям: Dictionary, List)
    10. Нет `?.Invoke()` с делегатами, захватывающими this (однократная аллокация при Connect допустима)

    Если найдены проблемы -- исправить:
    - LINQ -> for-цикл
    - Lambda в hot path -> instance method
    - new List/Array -> предаллоцированный буфер
    - String interpolation -> удалить или вынести за hot path
    - Boxing (передача value type как object) -> generic method

    Все `[MethodImpl(AggressiveInlining)]` на месте для: GetItemOffset, GetItemHeight, FindFirstVisibleIndex, Get, Release, RubberDelta, CalculateOffset, OnScrollPositionChanged.
  </action>
  <verify>
    <automated>grep -rn "new \|\.Select\|\.Where\|\.ToList\|\.ToArray\|\.OrderBy\|\.GroupBy" Runtime/Core/VirtualScroll/ Runtime/Core/Bindings/VirtualCollectionBinding.cs | grep -v "new()" | grep -v "new Stack\|new Dictionary\|new List\|new float\[" | grep -v "// alloc:" || echo "PASS: no unexpected allocs"</automated>
  </verify>
  <acceptance_criteria>
    - Нет LINQ-вызовов (.Select, .Where, .ToList, .ToArray, .OrderBy, .GroupBy) в Runtime/Core/VirtualScroll/ и VirtualCollectionBinding.cs
    - Нет `new` в LateUpdate, OnDrag, OnScroll, UpdateVisibleRange (кроме struct конструкторов)
    - FindVisibleRange возвращает struct VisibleRange (не class)
    - _indicesToRemove или аналогичный буфер аллоцируется в Activate/Connect, не в UpdateVisibleRange
    - AggressiveInlining присутствует на GetItemOffset, GetItemHeight, Get (pool), Release (pool)
  </acceptance_criteria>
  <done>Code review подтверждает zero-alloc в hot path. Все LINQ/new/boxing/lambda проблемы исправлены или отсутствуют.</done>
</task>
